Module 1

  1. Introduction au PL/SQL
    1. Ecrivez un programme affectant les valeurs 1 et 2 à deux variables a et b, puis permutant les valeurs de ces deux variables.
    2. Ecrivez un programme plaçant la valeur 10 dans une variable a, puis affichant la factorielle de a.
    3. Ecrivez un programme plaçant les valeurs 48 et 84 dans deux variables a et b puis affichant le pgcd de a et b.
  2. Tableaux et Structures
    1. Créez un type tableau pouvant contenir des entiers.
      * Créez une variable de ce type
      * Placez dans ce tableau la liste des 20 premiers carres parfaits : 1; 4; 9; 16; 25;.....
      * Inversez l’ordre des éléments du tableau
      * Affichez le tableau.
    2. Triez le tableau précédent avec la méthode du tri à bulle.
    3. Recherchez, par dichotomie, si l’élément 225 se trouve dans le tableau.
  3. Utilisation PL/SQL

    4. (Nous travaillerons sur les données A.6 et A.5. Vous n’oublierez pas de placer des commit en des lieux bien choisis. )

    1. Vous remarquerez que les valeurs des numpers de la table PERSONNE forment une s´séquence de nombres de 1 à 21.
      Utilisez une boucle dans laquelle vous placerez une requête pour recopier les couples nom/prénom de la table personne dans la table CLIENT.
    2. Ecrivez un script récupérant le client de clé primaire la plus élevée, et injectant ce client dans la table PERSONNEL.
    3. Ouvrez un compte courant pour chaque personne, effectuez un d´dépôt en espèce égal à numpers ∗ 100 euros.
    4. Ouvrez un livret pour chaque personne ayant un numpers pair, faites un virement de leur compte courant vers ce livret de sorte qu’il ne reste plus que 500 sur leur compte.
  4. Exceptions

    5. Nous utiliserons les données de A.7 et A.5 Vous êtes invités à modifier le code de la séance précédente. Chaque fois qu’un SELECT ... INTO ... sera effectué, vous rattraperez les exceptions NO DATA FOUND et TOO MANY ROWS. A chaque insertion, vous rattraperez l’exception DUP VAL ON INDEX.

    1. Faites de sorte que les scripts important les données des tables CLIENT ne puissent être exécutés qu’une seule fois.
    2. Les scripts remplissant la table Opération ne fonctionneront pas aujourd’hui... Même s’il fonctionnaient la dernière fois. Trouvez les codes d’erreurs des exceptions levées par ces scripts, rattrapez-les de la façon la plus appropriée qui soit.
